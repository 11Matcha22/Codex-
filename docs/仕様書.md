# Hook Swing Web版 仕様書

## 1. 対応環境
- 優先: スマートフォンブラウザ。
- 併用: PCブラウザ。
- 画面向き: 横固定前提（16:9キャンバス基準）。

## 2. 操作仕様
- 入力は1系統のみ。
  - 押下: フック接続を試行。
  - リリース: フック解除。
- 方向キーや複数ボタンは不使用。

## 3. ゲームループ
1. ステージ開始。
2. プレイヤーがスイング移動。
3. 成功: ゴールライン接触でクリア。
4. 失敗: 失敗ゾーン接触で失敗。
5. 即リトライ（失敗SEは待機不要）。

## 4. ステージ仕様
- 総数: 50（固定ステージ）。
- 解放: クリアで次ステージ解放。
- 到達済み定義: ステージ開始時点で到達済みに登録。
- 再プレイ: 到達済みステージはいつでも可。

## 5. 失敗・クリア判定
- 失敗: 画面下側の失敗ゾーンにプレイヤーが触れたとき。
- 画面外猶予: 失敗ゾーン下にマージンを設け、即死を避ける。
- クリア: ゴールラインに接触したとき。

## 6. UI仕様
- タイトル画面。
  - スタート。
  - ステージセレクト遷移。
  - 到達数、クリア数、合計ベスト表示。
- ステージセレクト。
  - 到達済みのみ選択可。
  - クリア済みを視覚表示。
- ゲーム画面。
  - ステージ番号、タイマー、メニューボタン。

## 7. 保存仕様（localStorage）
- reachedMax: 到達済み最大ステージ番号。
- cleared: クリア済みステージ辞書。
- bestTimes: ステージ別ベストタイム辞書。
- totalBest: 全50ステージのベスト合計（表示時に算出）。

## 8. サウンド仕様
- 失敗時SEのみ初期実装。
- SE再生は非同期で、ゲーム再開をブロックしない。

## 9. パフォーマンス方針
- Canvas 2Dで軽量描画。
- 固定タイムステップ更新。
- シンプル図形中心で60fpsを目標。
