# Hook Swing Web版 企画書

## 1. 目的
- Stickman Hookライクな「ワン入力で遊べる高速リトライ型アクション」を、スマホ優先のHTMLゲームとして実現する。
- 学習コストを抑え、反復挑戦で上達と爽快感を得られる体験を提供する。

## 2. ターゲット
- スマホで短時間プレイしたいライトユーザー。
- 1ステージ30秒前後の短い挑戦を繰り返したいユーザー。

## 3. 提供価値
- 押下/リリースだけで成立する直感操作。
- スイング連鎖で速度が乗る快感。
- 失敗→即再挑戦の短いゲームループ。

## 4. コア体験
- 押下中は近いフックポイントへ接続、リリースで解除。
- 振り子運動と慣性で前進し、ゴールライン接触でクリア。
- 画面下部の失敗ゾーン接触で失敗、SEを鳴らしつつ即リトライ。

## 5. スコープ（初期実装）
- 固定50ステージ。
- タイトル画面・ステージセレクト・ゲーム画面。
- 到達済みステージ再プレイ、クリアで次ステージ解放。
- localStorage保存（到達進行、クリア数、各ステージベスト、全50合計ベスト）。

## 6. 将来拡張
- 自動生成ステージ。
- ステージ作成モード（配置しながら即テスト）。
- 演出強化（エフェクト、カメラ演出、スキン拡張）。
